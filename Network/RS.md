R&S 所有的协议，本质上都在做同一件事——控制和转发数据流量。它们之间有着极强的共性和底层逻辑的关联。因此其实RS中所涉及的所有协议都可以按照下述的模型进行学习：

|   (协议报文、状态机、邻居建立、算法计算、数据库同步)      |     (大脑思考)
+-------------------------------------------------+
                        |
                        v 下发/映射
+-------------------------------------------------+
|   2. 转发面 (Data/Forwarding Plane)             | --> 最终落到硬件芯片上，数据包怎么走？
|   (CAM表、FIB前缀树、LFIB标签表、数据封装/剥离)        |     (肌肉执行)
+-------------------------------------------------+

|   (BFD毫秒级联动、GR不中断转发、Dot1x/MAB接入控制)   |     (免疫系统)
+-------------------------------------------------+

对此，我简单整理了RS中常见的，工作中最可能用到的协议，如下：
## 📐 一、 二层交换与拓扑控制家族 (Layer 2 Switching)
*   **STP / RSTP / MSTP (生成树协议族)**
    *   ├─ **控制面**：交互 BPDU，选举 Root/Designated/Root Port，计算无环拓扑，转换端口状态
    *   ├─ **转发面**：基于 CAM 表 (MAC地址表) 进行精确单播转发、未知单播/广播泛洪或端口阻塞 (Blocking)
    *   └─ **维护面**：BPDU Guard, Root Guard, Loop Guard, TC-BPDU 泛洪保护机制
*   **LACP / Eth-Trunk (链路聚合)**
    *   ├─ **控制面**：交互 LACPDU 报文，协商系统优先级、接口权重，维护活跃链路成员状态
    *   └─ **转发面**：物理多接口绑定，基于数据帧的 Hash 算法 (Source-Destination IP/MAC) 实现线速负载均衡
*   **VLAN / QinQ (虚拟局域网与双层标签)**
    *   ├─ **控制面**：GVRP/VTP (历史/特定场景) 动态同步 VLAN 数据库；划分广播域边界
    *   └─ **转发面**：在以太网帧中插入/剥离 802.1Q Tag (或 QinQ 双层 Tag)，在硬件层面实现二层逻辑隔离转发

## 🛣️ 二、 三层单播路由家族 (Layer 3 Unicast Routing)
*   **OSPF (开放式最短路径优先)**
    *   ├─ **控制面**：Hello 报文建邻 ➔ 8种状态机演进 ➔ 泛洪 1/2/3/4/5/7 类 LSA 同步 LSDB ➔ 运行 SPF 算法算路
    *   ├─ **转发面**：最优路径下发至 RIB ➔ 转换成 FIB (前缀树) 进行硬件级最长前缀匹配 (LPM) 转发
    *   └─ **维护面**：OSPF 与 BFD 毫秒级联动、GR (Graceful Restart) 不中断转发、区域防环机制 (骨干区域要求)
*   **EIGRP (加强型内部网关路由协议)**
    *   ├─ **控制面**：Hello 建邻 ➔ 交互 Update 同步拓扑表 ➔ 基于 5 个 K 值计算 Metric ➔ 运行 DUAL 算法 (FC 可行性条件验证)
    *   ├─ **转发面**：计算出 Successor (主) 和 Feasible Successor (备) 下发 FIB，支持等价/非等价 (Variance) 负载均衡
    *   └─ **维护面**：RTP (可靠传输协议) 序列号确认机制、SIA (卡在活动状态) 定时器与查询/应答防环机制
*   **BGP-4 (边界网关协议)**
    *   ├─ **控制面**：TCP 179 建邻 ➔ 6种状态机 ➔ 传递 NLRI ➔ 属性染色 (LP, MED, AS_Path, Community) ➔ 13条选路原则计算
    *   ├─ **转发面**：BGP 路由表优选后下发 FIB；在多主体场景下通过下一跳递归查询 (Recursive Lookup) 指向实际出接口
    *   └─ **维护面**：Route Reflector (RR) / Confederation 防环机制、BGP Peer Tracking、BFD 联动、Prefix-List/Route-Policy 强控
*   **Static Route & PBR (静态路由与策略路由)**
    *   ├─ **控制面**：人工静态指定下一跳/出接口，或者通过 Policy-Based Routing (Match Claues / Set Clauses) 编写硬策略
    *   └─ **转发面**：**PBR 会直接跳过标准 FIB 路由表**，强行修改硬件转发决策中的下一跳 IP、出接口或流量服务等级 (ToS)

## 📡 三、 多播与组播家族 (Multicast Network)
*   **IGMP v1/v2/v3 & MLD (成员关系协议)**
    *   ├─ **控制面**：主机与第一跳路由器交互 (Query / Report / Leave)，动态维护组播组成员关系表
    *   └─ **转发面**：结合二层 **IGMP Snooping**，拦截组播泛洪，在交换机芯片上精确将组播帧转发至特定加组端口
*   **PIM-SM / PIM-SSM (协议无关组播-稀疏/特定源)**
    *   ├─ **控制面**：邻居发现 ➔ RPF (反向路径转发) 检查防环 ➔ 构建共享树 (RPT) 与源树 (SPT) ➔ RP 选举 (BSR / Auto-RP)
    *   └─ **转发面**：生成 `(*, G)` 或 `(S, G)` 路由表项，映射至硬件**出接口列表 (OIF List)**，执行高效的数据包“一变多”复制转发
*   **MSDP (组播源发现协议)**
    *   ├─ **控制面**：跨 PIM 域交互，在不同的 RP (汇聚点) 之间通过 SA (Source Active) 报文共享/同步组播源信息
    *   └─ **转发面**：纯控制面协议，不直接参与组播流量转发，用于解决域间组播源可见性问题

## 🛡️ 四、 广域网、隧道与隔离家族 (WAN & VPN Tunneling)
*   **MPLS / LDP (多协议标签交换)**
    *   ├─ **控制面**：LDP 协议基于 IGP 单播路由表，为每个网络前缀分发公网传输标签，构建 LIB (标签数据库)
    *   └─ **转发面**：**放弃 IP 查表**，直接查询 **LFIB (标签转发信息表)**，在硬件层面进行标签的 Push (压入)、Swap (置换)、Pop (弹出)
*   **MP-BGP / MPLS VPN (多协议BGP虚拟专用网)**
    *   ├─ **控制面**：VRF 隔离路由 ➔ RD 转换为全球唯一 VPNv4 路由 ➔ RT 控制导入导出 ➔ MP-BGP 传路由并分发**内层私网标签**
    *   └─ **转发面**：PE 设备执行**双层标签嵌套**：外层标签跑公网 LDP 隧道 (传输)，内层标签在对端 PE 解开并扔进指定 VRF 转发
*   **VXLAN & BGP EVPN (虚拟扩展局域网)**
    *   ├─ **控制面**：BGP EVPN (Type 2/3/5 路由) 传递大二层 MAC 可达性、IP 路由及 VTEP 对端地址，动态建立隧道映射
    *   └─ **转发面**：将原始二层以太网数据帧，在 VTEP 芯片处直接封装进 **UDP 报文 (目的端口 4789)** 之中，在三层网络上打隧道传输
*   **GRE / DMVPN (动态多点VPN)**
    *   ├─ **控制面**：通过 NHRP (下跳解析协议) 动态解析分支机构 (Spoke) 的公网 IP 与私网 Tunnel IP 的映射关系
    *   └─ **转发面**：通用的点到点/点到多点隧道封装，将原始 IP 报文直接封装进新的 GRE 头部与公网 IP 头部 (Protocol 47) 跨网传输

## 🏥 五、 基础设施、高可用与安全联动 (Infrastructure & Safeguards)
*   **BFD (双向转发检测)**
    *   ├─ **控制面**：纯粹的独立链路状态监控协议。快速交互超轻量级 Hello 报文 (毫秒级)
    *   └─ **联动面**：自身不转发业务流量，一旦感知断线，**秒级/毫秒级通知** OSPF, BGP, Static Route 或 VRRP 控制面强制切换
*   **802.1X & MAB (接入层准入控制)**
    *   ├─ **控制面**：通过 EAPOL 状态机与 RADIUS/ISE 服务器进行身份/数字证书校验；若客户端无 Dot1x 驱动则触发 MAB (MAC 认证绕过)
    *   └─ **联动面**：在控制面校验通过前，通过硬件 ACL 在转发面将该端口阻断 (仅放行认证流量)，认证通过后下发 Dynamic VLAN/ACL 动态开通
