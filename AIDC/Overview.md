# AI算力中心导论
AI训练网络不是传统的数据中心网络

大多数传统数据中心的目标：
- 支撑web服务
- 支撑数据库系统
- 支撑微服务架构

以上应用的特点：
- 计算相对分散
- 流量以南北向为主
- 网络延迟要求不高

在大模型训练过程中：
- 数千张GPU同时参与计算
- 每一轮训练都需要进行梯度同步
- GPU之间需要持续交换大量数据

这导致了网络不再只是基础设施，而是决定训练效率的关键因素。

## 为什么AI训练需要高速网络
在深度学习训练过程中，每一次迭代都会执行以下步骤：
- 前向计算
- 反向传播
- 梯度同步
  
其中梯度同步需要在所有GPU之间交换数据。

当GPU数量增加时：
- 同步数据量迅速增加
- 通信时间可能超过计算时间

因此如果网络性能不足，训练速度会显著下降。

## AI训练的通信模式
常见的通信模式：
- AllReduce
- AllGather
- ReduceScatter
- Broadcast
- All-to-All
- 
其中，AllReduce是最核心的通信操作。它将每个GPU计算出的梯度进行汇总，并同步到所有GPU。

## 为什么网络会成为瓶颈
当GPU计算能力不断提升时，训练系统中的瓶颈也随之变化。
早期瓶颈在CPU、存储；如今GPU的算力已经飞速发展，单卡算力达到数百TFLOPS，此时系统瓶颈逐渐转移到GPU之间的数据通信。

## AI基础设施技术栈
一个完整的AI基础设施通常包含多个层次：
- 计算层
  - GPU
  - CPU
- 互联层
  - PCIe
  - NVLink
- 网络层
  - RDMA
  - RoCE
- 通信框架
  - NCCL
- 调度系统
  - Kubernetes
  - Slurm
