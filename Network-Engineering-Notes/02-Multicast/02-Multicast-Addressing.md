# 02-Multicast-Addressing.md (组播寻址与地址空间划分)

## 1. 组播 IP 地址的本质与传输特性

### 1.1 组播寻址的基本逻辑
* **目的地址属性**：组播 IP 地址在报文头部中**只能作为目的 IP（Destination IP）**。组播报文的源 IP（Source IP）永远是发送端设备的单播 IP 地址。
* **无状态性**：组播 IP 地址不属于任何具体的物理网卡或网络接口，它在网络中代表一个逻辑上的“数据频道”。

### 1.2 二进制特征与基本范围
* **Class D（D类地址）空间**：IPv4 组播地址固定使用 D 类地址空间，其地址段前缀表示为 `224.0.0.0/4`。
* **前缀特征**：字节流的最高 4 位固定为二进制的 `1110`。
* **十进制范围**：`224.0.0.0` 至 `239.255.255.255`。

## 2. IANA 组播地址空间细分与量化生产应用

IANA 对 D 类地址进行了严格的功能块划分，不同区块在交换机硬件和路由控制面上的处理行为截然不同：

### 2.1 局部网络控制块 (Local Network Control Block)
* **地址范围**：`224.0.0.0/24` （`224.0.0.0` - `224.0.0.255`）
* **转发特性**：**绝对不可路由**。网络设备在产生或转发此类报文时，其 IP 头部的 TTL（生存时间）值默认强制为 **1**(防止意外环路泛洪)。而三层设备在收到目的地址为224.0.0.0/24的组播报文时，也绝对不会将其向前转发（无条件本地丢弃），这意味着流量无法跨越三层接口，仅在当前二层 VLAN 内部流动。这是RFC规定的“链路本地组(Link-Local Groups)”硬性过滤规则。
* **标准协议预留实例**：
  * `224.0.0.1`：网段内所有多播主机（All Systems）。
  * `224.0.0.2`：网段内所有多播路由器（All Routers）。
  * `224.0.0.5` / `224.0.0.6`：OSPF 路由协议的核心报文交互地址。
  * `224.0.0.13`：PIM 组播路由协议的邻居发现（Hello）地址。

### 2.2 互联网控制与全局范围块 (Internetwork Control Block & GLOP)
* **地址范围**：`224.0.1.0` - `238.255.255.255`（排除特定的 SSM 区块）
* **转发特性**：属于公网可路由的全球范围组播地址。
* **典型实例与应用**：
  * `224.0.1.1`：预留给 NTP（网络时间协议）进行时钟同步组播。
  * **GLOP 地址空间 (`233.0.0.0/8`)**：这是一种分配机制，允许拥有公网 ASN（自治系统号）的机构，通过将 16 位的 ASN 填入组播 IP 的第二和第三字节，从而获得全球唯一的公网组播地址。

### 2.3 特定源组播区块 (SSM Block) —— 量化架构核心
* **地址范围**：`232.0.0.0/8` （`232.0.0.0` - `232.255.255.255`）
* **转发特性**：网络设备看到目的地在此范围内的报文时，**直接忽略 RP（汇聚点）控制面**，强制检查报文的已知单播源 IP，直接构建 `(S, G)` 最短路径树（SPT）。这是高频交易行情分发最青睐的极低延迟地址段。

### 2.4 本地管理组播区块 (Administratively Scoped Block) —— 内部流控核心
* **地址范围**：`239.0.0.0/8` （`239.0.0.0` - `239.255.255.255`）
* **转发特性**：类似于单播中的私有地址（如 `10.0.0.0/8`）。公网路由器会无条件丢弃此范围的流量，专门用于企业内部、数据中心内部的组播业务隔离。
* **量化实战应用**：用于量化 company 内部的私有数据分发，如交易母单策略向内部风控模块、本地历史回测组件分发实时的流数据。

## 3. 组播寻址策略配置与表项验证

### 3.1 查验交换机默认的 SSM 地址映射范围
在现代网络设备中，可以通过控制台命令查看系统当前的特定源组播地址策略（以思科设备为例）：

```text
Switch# show ip pim ssm
PIM SSM status : Enabled
Current SSM range : 232.0.0.0/8
```
### 3.2 生产环境中的 SSM 地址范围裁剪与特化配置
在实际的量化测试或特定机房对接中，若外部行情的组播地址由于历史原因没有规划在 `232.0.0.0/8` 内，工程师需要通过标准 ACL 强行将非标准地址段定义为 SSM 行为，以跳过 RP 疏导协商：

```text
! 定义需要强制走 SSM 的目标地址范围
Switch(config)# ip access-list standard QUANT-SSM-EXT
Switch(config-std-nacl)# permit 239.100.0.0 0.0.255.255
Switch(config-std-nacl)# exit

! 覆盖设备默认的 232/8 策略，使指定地址段同样具备无 RP 的硬件转发特性
Switch(config)# ip pim ssm range QUANT-SSM-EXT
```

## 4. 组播网络寻址设计的工程直觉

### 4.1 地址前缀决定控制面状态机制
面对任何组播 IP，第一直觉应当是通过其前缀快速判定其生存域与技术栈：
* 命中 `224.0.0.X`：直觉反应为生存时间限制（TTL = 1），流量绝对无法跨越三层路由接口，排查故障时无需考虑三层 PIM 邻居状态。
* 命中 `232.X.X.X`：直觉反应为纯粹的 `(S, G)` 硬件转发，控制面上绝不存在 RP 选举、汇聚点单点收敛或 Anycast-RP 同步延迟的问题。

### 4.2 严格内外部地址边界审查
在量化机房边界（Hand-off 端口），必须对地址空间进行严格的安全过滤。严禁任何带有 `239.0.0.0/8` 本地私有前缀的流量流出到交易所网络，同时严禁非授权的 `224.0.0.X` 协议报文注入内部生产网，防止扰乱本地交换机的二层控制面拓扑。
